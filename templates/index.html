Описание проекта

В базе хранится информация о сервисах и их состояниях - сервис может быть работающим, неработающим, либо может работать, но нестабильно.

Сайт

С помощью меню в левой части экрана возможно:

  • Посмотреть список действующих сервисов и их текущие состояния
  • Посмотреть историю изменений состояний по интересующему сервису
  • Рассчитать SLA по интересующему сервису и время его недоступности за заданный период
  • Добавить информацию о работе сервисов

API

  • POST-запрос по пути '/add_info/api' добавляет информацию в БД. Поля запроса:
    1. "service":"Название существующего или нового сервиса"
    2. "state":"(Работает / Не работает / Работает нестабильно)"
    3. "description":"Любая информация"
  • GET-запрос по пути '/services/api' возвращает список отслеживаемых сервисов с их текущими состояниями
  • POST-запрос по пути '/state_history/api' возвращает историю изменения состояний по сервису.
    Поле запроса: "service":"Название существующего сервиса"
  • POST-запрос по пути '/sla/api' возвращает показатель SLA и время, которое сервис не работал в заданном интервале. Поля запроса:
    1. "service":"Название существующего сервиса"
    2. "start":"Дата и время в формате %Y-%m-%d %H:%M"
    3. "end":"Дата и время в формате %Y-%m-%d %H:%M"
